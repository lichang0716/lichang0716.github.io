Twitter Login

发表于 2017-06-21   |   分类于 iOS   |  

开发者网址：dev.twitter.com

准备

首先需要注册 twitter 开发者账号，

之后进入 apps.twitter.com 创建应用。

创建应用必须要填写 Name，Description，Website 三项。

关于 Callback URL ：

测试当通过 App 进行授权登录时，可以为空；

当通过网页授权登录时，必须要填写 URL，否则会提示下面所示的错误：

[TwitterKit] did encounter error with message "Error obtaining user auth token."

创建应用后，在 Keys and Access Tokens 页面获取 Consumer Key 和 Consumer Secret。

工程

配置工程：twitterkit iOS installation

  • 在项目中导入 iOS SDK
  • 在项目 App Delegate 中初始化 Twitter 类
  • 在 info.plist 中配置 URLSchemes 和 LSApplicationQueriesSchemes
  • 如果使用网页登录，需要添加 SafariServices.framework

登录步骤：twitterkit iOS login

  • 在 ViewController 中添加 TWTRLogInButton 或者自定义按钮添加 logInWithCompletion 动作

  • 在 -application:openURL:options: 中实现 -twitter:openURL:options 方法，

    此方法会在返回自己应用的时候，对 token 进行保存操作。

登录成功后，在回调里的 TWTRSession 对象中获取 token，username，userid 等信息。

Demo 地址：Twitter Login - Swift

移除项目中的 CocoaPods

发表于 2016-10-26   |   分类于 iOS   |  

关于如何移除 iOS 项目中使用的 CocoaPods，这类问题可以 Google 出不少答案。

比如：How to remove CocoaPods from a project? - Stack Overflow.

这里自己再整理一遍.

当前环境：

Project: Objective-C
Xcode: 8.1 GM
Pod: 1.0.0.beta.3

步骤：

  • 删除项目文件夹中的 Podfile , Podfile.lock 和 Pods 文件夹；
  • 删除生成的 .xcworkspce 文件
  • 从 .xcodeproj 打开项目，删除对应 targets -> General -> Linked Frameworks and Libraries -> libPods.a
  • 在对应 targets -> Build Phases，删除 Copy Pods Resources，Embed Pods Frameworks 以及 Check Pods Manifest.lock

到这里操作基本完成。

但是在我的项目中此时出现了大量的 Warning，大都提示为 missing file 之类。

后来发现原因是因为使用了 git， 这时对项目进行一次 commit 操作即可修复这些警告。

同时发现了一个插件，应该可以取代部分手动操作，还没有实验，这里先贴出来地址：

cocoapods-deintegrate - GitHub

有兴趣可以试一下。

iOS 10 中的其中一个改变是对于隐私数据的保护。

You must statically declare your app’s intended use of protected data classes 
by including the appropriate purpose string keys in your Info.plist file.

大意为：你必须在 app 中的 info.plist 声明一个 string key 来表达你想使用受保护数据的目的。

After Apple 2016 Fall Event

发表于 2016-09-08   |   分类于 something   |  

昨晚看了 2016 Apple 的新品发布会。
本来最想看到的是 Mac 新品，因为最近有这个需求，但最后发现没有…sad.
只有 iPhone, apple watch 和 AirPods。
之后有些想法如下：

Courage

这是在介绍取消旧耳机插孔时，speaker 提到的一个词，总感觉有一种莫名其妙的硬气在里面。
个人认为取消耳机孔这个点是能让 iPhone7/plus 称为一款新产品的原因之一。
（另一个我认为是新 Home 键，但是当前的讨论热度相比之下明显集中在耳机上）
并且在当前 3.5mm 耳机正在大量使用的情况下，apple 弃用并改为 Lightning 确实勇气可嘉。
也许从一开始就是 Apple 给什么，用户用什么。

Creative

虽然在发布会之前已经露出来了大部分要发布的信息，但是完整看下来并不觉得无聊。
就比如事先知道了一部电影的概要，但是在看的过程中还是会为一些片段感到惊喜。
一些 已知的 feature 从 speakers 口中讲出来的时候，还是会感觉有意思，
比如 watch 的防水喇叭，iPhone 的 Home 键等等。
回到 Apple 的创新创造力上，这是应该一个既客观又主观的东西。
但也许就是不关注不看发布会的人可能打 10 分；
关注这些也看了发布会的人打 20 分；
Apple 给自己打 30 分甚至更多这样吧。

Joke

在观看这次发布会中以及在微博和推上也看到一些有意思的地方，逗人一笑，比如：
a. Everyone can code…except Fenng.
b. Just Do IT Sunday.
c. The Biggest iOS release ever. (16G Devices：Σ( ° △ °|||)︴)

Summary

其实总的来看这次 Apple 的发布会还是挺有意思的，并没有特别失望的感觉。
下次发布会见，得有新款 Mac 了吧！
