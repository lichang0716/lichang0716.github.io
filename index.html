一些想法

发表于 2018-07-28   |   分类于 something   |  

最近几天事情比较多，一周时间有爆发的疫苗事件，大陆 METOO，虎扑撕 WYF，北京美国大使馆爆炸…社会焦点似乎每天都在进行转变。这其中的缘由不去想，单看每个事件也是很不少想要说的。

信任

信任这个东西，是从今年六月份就一直开始想的一个东西。在我眼中，目前大陆范围内有利益冲突的双方似乎没有任何信任而言。

这种情况并不是毫无缘由，也许先是市场上的商品以次充好，消费者想占便宜反倒吃了亏，之后消费者不想占便宜，还是还买假的东西，甚至消费者买到货真价实的东西，仍然觉得自己在其他方面还是吃了亏。于是在生活中我们各种感觉自己被坑，买鞋可能是假的，去饭馆吃饭肉是假的油是假的，看病怀疑医生多开药，买房也怀疑开发商售楼处隐瞒消息骗人，总之那些本应该是「解决问题」的地方，似乎都在解决问题的过程中存在着「偷工减料」的行为。你想挣的多，我想花的少，问题也就出现了。

现在疫苗也出现了问题，人们也许对药企，对医疗体系，对政府的监管过程也更加不信任，毕竟这次很多人都被坑了，并且坑的都是自己以及自己的心头肉。信任感越来越低，问题就会越来越大。可疫苗问题到现在还没有解决，关注度也渐渐下降，好了伤疤忘了疼好想已经不足以形容中国社会问题了。也许是疫苗仍然影响的是一部分人，而不是全部的人？

前进后退

紧接着疫苗事件的是中国的 METOO 行动，各个领域相继爆出性侵猥亵性骚扰女性的事件。其实女性作为现代社会一半组成部分，能力和意识同若干年前肯定是进步了许多，这次行动也可以看作顺应时代发展而出现的，当然带头的人仍是勇敢的，有人作出不一样的事情，发出不一样的声音，社会也许才能往更好的方向发展。

希望更多问题的出现是在推动着社会的进步，而不是进了几步之后又踏回来，即使社会踏了回来，也希望可以让更多人分清什么是好和不好。

Skr Skr

虎扑和 WYF 的事情，笑了我半个早上。有实力的人更容易被社会接受，超前和艺术性的东西现在不被认可再正常不过，争辩毫无意义，留给时间去验证好了。真正让我感觉到无奈的是很多人的认知和喜好是被引导和操控的，所以我特别讨厌「热门微博」这个东西，希望大家有时间还是多看看书吧Skr。

假象

北京美国大使馆发生爆炸，「疑似爆竹」的表达方式似乎也没有完全隐瞒事情真相，但总是总有一种故意营造和平时代的感觉。

Universal Link 使用体会

发表于 2018-03-09   |   分类于 iOS   |  

最近在公司接了「通用链接」，把使用心得记录一下。

通用链接优于 Scheme 的部分：

1.Scheme 不具有唯一性，即多个 App 可能恰巧使用了相同的 Scheme，但是通用链接有唯一性;

2.使用 Scheme 跳转，在没有相应安装 App 时，有「Safari 打不开该网页，因为网址无效」的错误提示；

3.有的App 禁用了 Scheme。

通用链接使用：

使用方法不详细记录，Support Universal Links - Apple 官网文档写的很详细

需要注意的地方：

1.跨域跨域跨域

这是非常需要注意的地方。

例如知乎的分享页面地址是「www.zhihu.com/…」,而「打开 App」按钮的对应地址为「oia.zhihu.com/…」;

同样网易云音乐的分享页面地址是「music.163.com/…」,跳转地址是「m.music.163.com」。

也就是说，只有跨域的时候，才能触发通用链接。

因此一个较为完整的通用链接的触发流程为：

a.分享页面（域名A）-> b.点击「打开 App」按钮 ->

​ c1.着陆页（域名B）-> d1.微信等需要提示浏览器打开

​ c2.如果是在浏览器，此时正常已经跳转到 App 内

2.保留 Scheme 跳转的方式

在使用过程中，发现「手机 QQ 浏览器」始终不会触发通用链接，所以这个时候，只能用 Scheme 跳转到 App。

1.在 4G 或者正常 Wi-Fi 不连接 Http 代理的情况下，安装 App 可以正常使用通用链接跳转，但是通过 Charles 在开发环境连调时，不会触发通用链接（没有使用通用连接对应域名的 Host）。

iOS 图片压缩剪裁

发表于 2017-10-10   |   分类于 iOS   |  

最近遇到一个面试题：如何把一个 1080P 的图片，修改为和设备屏幕宽度相同高度等比例的图片。当时没答上来，回来查了一下相关的知识点，记录一下。

压缩和剪切图片：

// 原始图片
NSData *imageData = [NSData dataWithContentsOfURL:[NSURL URLWithString:@"http://oersnre8d.bkt.clouddn.com/planet.jpg"]];
NSLog(@"原始 data length = %ld", imageData.length);
UIImage *realPlanetImage = [UIImage imageWithData:imageData];
NSLog(@"image = %@", realPlanetImage);
// 压缩
NSData *newData = UIImageJPEGRepresentation(realPlanetImage, 0.1);
NSLog(@"修改压缩系数 data length = %ld", newData.length);
NSLog(@"image = %@", [UIImage imageWithData:newData]);
// 裁剪
UIImage *newImage = [self image:realPlanetImage fortargetSize:CGSizeMake(self.view.bounds.size.width, (realPlanetImage.size.height / realPlanetImage.size.width) * self.view.bounds.size.width)];
NSData *newData2 = UIImageJPEGRepresentation(newImage, 1.0);
NSLog(@"裁剪之后 data length = %ld", newData2.length);
NSLog(@"image = %@", newImage);

打印结果为：

原始 data length = 42566
image = <UIImage: 0x170091940>, {1920, 1080}
修改压缩系数 data length = 35632
image = <UIImage: 0x1740933d0>, {1920, 1080}
裁剪之后 data length = 15377
image = <UIImage: 0x1740933d0>, {375, 211}

由此可见：

修改压缩系数，并不会改变图片的原始尺寸，即像素数不变，改变的是文件大小；

对图片进行裁剪，像素数减少，图片的大小和尺寸都后进行改变。

PS：

/**
 裁剪图片

 @param image 原始图片
 @param targetSize 目标大小
 @return 新的图片
 */
- (UIImage*)image:(UIImage*)image fortargetSize:(CGSize)targetSize {
    UIImage *sourceImage = image;
    // 目标宽度高度
    CGFloat targetWidth = targetSize.width;
    CGFloat targetHeight = targetSize.height;
    CGFloat scaledWidth = targetWidth * [UIScreen mainScreen].scale;
    CGFloat scaledHeight = targetHeight * [UIScreen mainScreen].scale;
    // 剪切起始位置
    CGPoint thumbnailPoint = CGPointMake(0.0, 0.0);
    // 开始剪切
    UIGraphicsBeginImageContext(targetSize);
    CGRect thumbnailRect = CGRectZero;
    thumbnailRect.origin= thumbnailPoint;
    thumbnailRect.size.width= scaledWidth;
    thumbnailRect.size.height= scaledHeight;
    [sourceImage drawInRect:thumbnailRect];
    // 拿到新图片
    UIImage *newImage = UIGraphicsGetImageFromCurrentImageContext();
    return newImage;
}

iOS 圆形头像

发表于 2017-09-29   |   分类于 iOS   |  

iOS 显示圆形头像是一个比较常见的需求，之前面试的时候也有被问到。当时答出来的是设置 CALayer 以及在 Storyboard 中设置，第二种没有答出来。现在把三种方法写一下，效果图如下：

CALayer

UIView 和 CALayer 的关系在这里不多说，CALayer 主要的作用是绘制。

目前假设一个 UIImageView 叫 avatarImageView，那么方法为：

// 设置 layer 以外的区域不显示，默认为 NO
avatarImageView.layer.masksToBounds = YES;
// 设置 layer 的圆角
avatarImageView.layer.cornerRadius = avatarImageView.bounds.size.height/2;
// 设置 layer 边框颜色和宽度
avatarImageView.layer.borderWidth = 3.0;
avatarImageView.layer.borderColor = [UIColor whiteColor].CGColor;

Duartz2D

直接提供方法：

/**
 获取圆形图片

 @param originImage 原图
 @param borderWidth 边框宽度
 @param borderColor 边框颜色
 @return 返回圆形图片
 */
- (UIImage *)getRoundAvatar:(UIImage *)originImage borderWidth:(CGFloat)borderWidth borderColor:(UIColor *)borderColor {
    // 定义新图片的尺寸
    CGFloat bigImageWidth = originImage.size.width + 2 * borderWidth;
    CGFloat bigImageHeight = originImage.size.height + 2 * borderWidth;
    CGSize imageSize = CGSizeMake(bigImageWidth, bigImageHeight);
    // 开启上下文
    UIGraphicsBeginImageContextWithOptions(imageSize, NO, 0.0);
    // 取得当前上下文
    CGContextRef ctx = UIGraphicsGetCurrentContext();
    // 设置边框颜色
    [borderColor set];
    // 设置边框
    CGFloat bigRadius = bigImageWidth / 2.0;
    CGFloat centerX = bigRadius;
    CGFloat centerY = bigRadius;
    CGContextAddArc(ctx, centerX, centerY, bigRadius, 0, M_PI * 2, 0);
    CGContextFillPath(ctx);
    // 设置小圆
    CGFloat smallRadius = bigRadius - borderWidth;
    CGContextAddArc(ctx, centerY, centerY, smallRadius, 0, M_PI * 2, 0);
    // 裁剪
    CGContextClip(ctx);
    // 画图
    [originImage drawInRect:CGRectMake(borderWidth, borderWidth, originImage.size.width, originImage.size.height)];
    // 获取新图片
    UIImage *newImage = UIGraphicsGetImageFromCurrentImageContext();
    // 结束上下文
    UIGraphicsEndImageContext();
    return newImage;
}

调用方式：

[avatarImageView setImage:[self getRoundAvatar:[UIImage imageNamed:@"somePic.jpg"]
                                   borderWidth:3.0
                                   borderColor:[UIColor whiteColor]]];

Twitter Login

发表于 2017-06-21   |   分类于 iOS   |  

开发者网址：dev.twitter.com

准备

首先需要注册 twitter 开发者账号，

之后进入 apps.twitter.com 创建应用。

创建应用必须要填写 Name，Description，Website 三项。

关于 Callback URL ：

测试当通过 App 进行授权登录时，可以为空；

当通过网页授权登录时，必须要填写 URL，否则会提示下面所示的错误：

[TwitterKit] did encounter error with message "Error obtaining user auth token."

创建应用后，在 Keys and Access Tokens 页面获取 Consumer Key 和 Consumer Secret。

工程

配置工程：twitterkit iOS installation

  • 在项目中导入 iOS SDK
  • 在项目 App Delegate 中初始化 Twitter 类
  • 在 info.plist 中配置 URLSchemes 和 LSApplicationQueriesSchemes
  • 如果使用网页登录，需要添加 SafariServices.framework

登录步骤：twitterkit iOS login

  • 在 ViewController 中添加 TWTRLogInButton 或者自定义按钮添加 logInWithCompletion 动作

  • 在 -application:openURL:options: 中实现 -twitter:openURL:options 方法，

    此方法会在返回自己应用的时候，对 token 进行保存操作。

登录成功后，在回调里的 TWTRSession 对象中获取 token，username，userid 等信息。

Demo 地址：Twitter Login - Swift

iOS View 添加多个手势

发表于 2017-03-10   |   分类于 iOS   |  

iOS Gesture (手势) 是一个比较常用的东西，在开发手势时需要用到 UIGestureRecognizer 类。

UIGestureRecognizer 继承自 NSObject。

它的子类有 6 个：

UITapGestureRecognizer          点击
UIPinchGestureRecognizer        缩放    
UIRotationGestureRecognizer        旋转
UISwipeGestureRecognizer        滑动
UIPanGestureRecognizer            拖动
UILongPressGestureRecognizer    长按

一个手势对象只可以作用于一个 View，但是一个 View 可以添加多个手势。

那么如何可以让多个手势同时作用于 View 呢？下面是个小例子。

假设有一个 UIImageView 对象：imageView（原谅这个命名…）

首先需要设置让 View 可以接受我们的事件和消息：

imageView.userInteractionEnabled = YES;

之后添加旋转和缩放手势：

// 添加缩放手势
UIPinchGestureRecognizer *pinchGesture = [[UIPinchGestureRecognizer alloc] initWithTarget:self action:@selector(handlePinch:)];
pinchGesture.delegate = self;
[imageView addGestureRecognizer:pinchGesture];
// 添加旋转手势
UIRotationGestureRecognizer *roationGesture = [[UIRotationGestureRecognizer alloc] initWithTarget:self action:@selector(handleRoation:)];
[imageView addGestureRecognizer:roationGesture];

要让一个 View 同时相应多个 Gesture ，需要实现 UIGestureRecognizerDelegate 中的:

- (BOOL)gestureRecognizer:(UIGestureRecognizer *)gestureRecognizer shouldRecognizeSimultaneouslyWithGestureRecognizer:(UIGestureRecognizer *)otherGestureRecognizer {
    // 设置允许支持多个手势，默认不支持
    return YES;
}

这样就可以用两个手指让图片同时旋转和缩放。

移除项目中的 CocoaPods

发表于 2016-10-26   |   分类于 iOS   |  

关于如何移除 iOS 项目中使用的 CocoaPods，这类问题可以 Google 出不少答案。

比如：How to remove CocoaPods from a project? - Stack Overflow.

这里自己再整理一遍.

当前环境：

Project: Objective-C
Xcode: 8.1 GM
Pod: 1.0.0.beta.3

步骤：

  • 删除项目文件夹中的 Podfile , Podfile.lock 和 Pods 文件夹；
  • 删除生成的 .xcworkspce 文件
  • 从 .xcodeproj 打开项目，删除对应 targets -> General -> Linked Frameworks and Libraries -> libPods.a
  • 在对应 targets -> Build Phases，删除 Copy Pods Resources，Embed Pods Frameworks 以及 Check Pods Manifest.lock

到这里操作基本完成。

但是在我的项目中此时出现了大量的 Warning，大都提示为 missing file 之类。

后来发现原因是因为使用了 git， 这时对项目进行一次 commit 操作即可修复这些警告。
