iOS 倒计时的简单实现

发表于 2016-09-17   |   分类于 iOS   |  

最近封装 UI 的时候，有一个倒计时的功能，直接用的是 NSTimer。

后来搜了下还有其他两种方法，这里记录一下（只是简单实现，实际应用需要完善）

NSTimer

计时器是应该是最常见，也是最常用的一种：

声明：

int timeDown;
NSTimer *timer;

调用：

timeDown = 59;
timer = [NSTimer scheduledTimerWithTimeInterval:1.0
                                         target:self
                                       selector:@selector(handleTimer)
                                       userInfo:nil
                                        repeats:YES];
[timer fire];

实现：

- (void)handleTimer {
    if(timeDown > 0) {
        _timerLabel.text = [NSString stringWithFormat:@"%d", timeDown];
    } else {
        _timerLabel.text = @"倒计时结束";
        [timer invalidate];
    }
    timeDown = timeDown - 1;
}

NSThread

NSThread 用到的是 performSelectorInBackground:withObject: 方法

声明：

int count;

调用：

[self performSelectorInBackground:@selector(threadCounting) withObject:nil];

实现：

- (void)threadCounting {
    for(int i = 59; i >= 0; i--) {
        count = i;
        // 更新 Label 需要在主线程执行
        [self performSelectorOnMainThread:@selector(mainThreadCounting)
                               withObject:nil
                            waitUntilDone:YES];
        sleep(1);
    }
}

- (void)mainThreadCounting {
    if (count > 0) {
        _timerLabel.text=[NSString stringWithFormat:@"%d",count];
    } else {
        _timerLabel.text = @"倒计时结束";
    }
}

GCD

GCD 中有一个定时器模式：

实现：

- (void)gcdCouting {
    __block int second = 59;
    // 默认优先级的全局队列
    dispatch_queue_t queue = dispatch_get_global_queue(DISPATCH_QUEUE_PRIORITY_DEFAULT, 0);
    // 定时器模式
    dispatch_source_t timer = dispatch_source_create(DISPATCH_SOURCE_TYPE_TIMER, 0, 0, queue);
    // 设置 1 秒间隔
    dispatch_source_set_timer(timer, DISPATCH_TIME_NOW, 1 * NSEC_PER_SEC, 0 * NSEC_PER_SEC);
    dispatch_source_set_event_handler(timer, ^{
        // 回到主线程更新 label
        dispatch_sync(dispatch_get_main_queue(), ^{
            if (second >= 0) {
                _timerLabel.text = [NSString stringWithFormat:@"%d", second];
                second --;
            } else {
                dispatch_source_cancel(timer);
                _timerLabel.text = @"倒计时结束";
            }
        });
    });
    dispatch_resume(timer);
}

以上就是三种常用的倒计时方法。

After Apple 2016 Fall Event

发表于 2016-09-08   |   分类于 something   |  

昨晚看了 2016 Apple 的新品发布会。
本来最想看到的是 Mac 新品，因为最近有这个需求，但最后发现没有…sad.
只有 iPhone, apple watch 和 AirPods。
之后有些想法如下：

Courage

这是在介绍取消旧耳机插孔时，speaker 提到的一个词，总感觉有一种莫名其妙的硬气在里面。
个人认为取消耳机孔这个点是能让 iPhone7/plus 称为一款新产品的原因之一。
（另一个我认为是新 Home 键，但是当前的讨论热度相比之下明显集中在耳机上）
并且在当前 3.5mm 耳机正在大量使用的情况下，apple 弃用并改为 Lightning 确实勇气可嘉。
也许从一开始就是 Apple 给什么，用户用什么。

Creative

虽然在发布会之前已经露出来了大部分要发布的信息，但是完整看下来并不觉得无聊。
就比如事先知道了一部电影的概要，但是在看的过程中还是会为一些片段感到惊喜。
一些 已知的 feature 从 speakers 口中讲出来的时候，还是会感觉有意思，
比如 watch 的防水喇叭，iPhone 的 Home 键等等。
回到 Apple 的创新创造力上，这是应该一个既客观又主观的东西。
但也许就是不关注不看发布会的人可能打 10 分；
关注这些也看了发布会的人打 20 分；
Apple 给自己打 30 分甚至更多这样吧。

Joke

在观看这次发布会中以及在微博和推上也看到一些有意思的地方，逗人一笑，比如：
a. Everyone can code…except Fenng.
b. Just Do IT Sunday.
c. The Biggest iOS release ever. (16G Devices：Σ( ° △ °|||)︴)

Summary

其实总的来看这次 Apple 的发布会还是挺有意思的，并没有特别失望的感觉。
下次发布会见，得有新款 Mac 了吧！
